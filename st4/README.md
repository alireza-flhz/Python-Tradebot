# st4 — Scalping: Support/Resistance + FVG (Indicator)

اندیکاتور اسکالپ برای تایم‌فریم‌های **1m / 5m / 30m**. فقط روی چارت **رسم می‌کنه**
و سفارش نمی‌ذاره (نسخهٔ Indicator).

## چی روی چارت می‌بینی

| المان | توضیح |
|-------|-------|
| **حمایت/مقاومت مهم (S/R)** | فقط سطوحی که چند بار لمس شدن (پیوت‌های نزدیک هم ادغام و تعداد تاچ شمرده می‌شه) |
| **FVG مهم** (Fair Value Gap) | فقط گپ‌های به‌اندازهٔ کافی بزرگ؛ گپ‌های پرشده حذف می‌شن |
| **پیشنهاد ورود (BUY/SELL)** | فلش + برچسب با **Entry / SL / TP** و خطوط چین حد ضرر و حد سود |
| **پس‌زمینهٔ کم‌رنگ** | جهت مجاز معامله بر اساس بایاس 30m |

## نقطهٔ ورود چطور پیشنهاد داده می‌شه (کانفلوئنس)

پیشنهاد ورود فقط وقتی صادر می‌شه که چند شرط با هم جور بشن — نه با یه اندیکاتور تنها:

**LONG (خرید):**
- بایاس 30m صعودی ✅
- قیمت به یه **حمایت مهم** واکنش نشون بده (لمسش کنه و بالاش بسته بشه)
- قیمت بالای EMA21 + هم‌جهت با VWAP + کندل صعودی + RSI بالای ۵۰
- **Entry:** قیمت بسته‌شدن کندل تأیید
- **Stop Loss:** کمی زیر همون حمایت (به اندازهٔ `SL buffer × ATR`)
- **Take Profit:** نزدیک‌ترین **مقاومت مهم** بالاتر؛ اگه نبود، بر اساس `Reward:Risk` (پیش‌فرض ۲)

**SHORT (فروش):** دقیقاً برعکس روی یه **مقاومت مهم**.

> برچسب اگه **`+FVG`** داشته باشه، یعنی قیمت داخل یه FVG هم‌جهت هم هست (کانفلوئنس قوی‌تر).

> ⚠️ خطوط **EMA و VWAP روی چارت کشیده نمی‌شن** (چارت تمیز). این مقادیر فقط پشت‌صحنه
> محاسبه می‌شن چون موتور سیگنال ازشون استفاده می‌کنه.

## حمایت و مقاومتِ «مهم» چطور تشخیص داده می‌شه
- با **پیوت سوئینگ**: سقف با `srLen` کندل پایین‌تر در دو طرف → مقاومت؛ کف → حمایت.
- پیوت‌های نزدیک هم (در فاصلهٔ `Merge tolerance × ATR`) **ادغام** می‌شن و به‌عنوان یک سطح شمرده می‌شن.
- هر بار قیمت دوباره اون ناحیه رو بزنه، **تعداد تاچ** بالا می‌ره.
- سطحی که تعداد تاچش `≥ Min touches` باشه **«مهم»** حساب می‌شه: خط پررنگ و توپُر با برچسب تعداد تاچ (مثلا `3x`).
- با گزینهٔ **Show only important**، سطوح کم‌اهمیت اصلاً نشون داده نمی‌شن.

## FVGِ «مهم» چطور تشخیص داده می‌شه
- **FVG صعودی:** کف کندل فعلی بالاتر از سقف دو کندل قبل (گپ) → باکس سبز.
- **FVG نزولی:** سقف کندل فعلی پایین‌تر از کف دو کندل قبل → باکس قرمز.
- فقط گپ‌هایی که اندازه‌شون `≥ Min FVG size × ATR` باشه رسم می‌شن (گپ‌های ریز/نویزی حذف).
- وقتی قیمت دوباره داخل گپ برگرده و پرش کنه (**mitigation**)، باکس **حذف** می‌شه — فقط FVGهای فعال و پرنشده روی چارت می‌مونن.

## نحوهٔ اجرا در TradingView
1. **Pine Editor** → کل `ScalpEmaVwapRsi.pine` رو پیست کن → **Add to chart**.
2. چارت رو بذار روی **1m یا 5m**. بایاس 30m خودکار اعمال می‌شه.
3. برای هشدار: راست‌کلیک روی سیگنال → **Add alert** (سیگنال، FVG صعودی و FVG نزولی همه Alert دارن).

## پارامترهای قابل تنظیم (منوی ⚙️)
- **Support/Resistance:** Pivot Lookback (۱۵)، **Min touches** برای مهم‌بودن (۲)، **Merge tolerance ×ATR** (۰.۵)، حداکثر سطوح (۱۰)، فقط سطوح مهم
- **FVG:** **Min FVG size ×ATR** (۰.۵)، حداکثر باکس (۱۲)، امتداد به راست (۱۵ کندل)، حذف گپ‌های پرشده
- **Importance:** طول ATR برای فیلترها (۱۴)
- **Entry suggestion:** رسم Entry/SL/TP، بافر حد ضرر (×ATR)، نسبت Reward:Risk، طول خطوط، تعداد پیشنهادهای نگه‌داشته‌شده
- **موتور سیگنال:** EMA Trend filter، RSI، اجبار VWAP، روشن/خاموش کردن پیشنهادها
- **Colors:** رنگ حمایت، مقاومت، FVG صعودی و نزولی

### تنظیم دقت (Tuning)
- سطوح خیلی زیاد؟ → **Min touches** رو ببر ۳، یا **Pivot Lookback** رو زیادتر کن.
- FVG زیاد و ریز؟ → **Min FVG size ×ATR** رو ببر بالاتر (مثلا ۰.۸ یا ۱.۰).
- سطوح تیکه‌تیکه؟ → **Merge tolerance ×ATR** رو زیادتر کن تا نزدیک‌ها بیشتر ادغام شن.

## نکات معاملاتی
- سطوح S/R محل مناسب برای حد ضرر/سود و انتظار واکنش قیمتن.
- FVGها اغلب به‌عنوان ناحیهٔ برگشت قیمت پر می‌شن؛ ورود نزدیک FVG هم‌جهت با بایاس قوی‌تره.
- فقط در جهت بایاس 30m (پس‌زمینهٔ سبز = Long، قرمز = Short) و در اورلپ لندن-نیویورک معامله کن.
